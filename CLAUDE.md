## Tech Stack

- **Next.js 14** (App Router) + **React 18** + **TypeScript 5.4** (strict mode)
- **d3 7.9** for map projections and SVG rendering (`geoOrthographic` for 3D globe, `geoNaturalEarth1` for 2D)
- **Zustand 4.5** for state management (single store in `src/stores/useStore.ts`)
- **Lucide React** for icons
- All styling is **inline React styles** — no Tailwind, no CSS modules. Theme tokens live in `src/lib/theme.ts`.

## Architecture

Single-page client-side app that displays aircraft, vessels, and trains on an interactive globe/map.

### Data Sources
- **Aircraft**: Live data from OpenSky Network REST API (`/api/states/all`), fetched every 15s by `useOpenSky` hook. Anonymous access, may be CORS-blocked in browser.
- **Vessels & Trains**: Simulated in-browser. Generated by `src/lib/generate.ts`, animated every 3s by `useSimulation` hook using great-circle interpolation (`src/lib/geo.ts`).

### Data Flow
`useOpenSky` / `useSimulation` hooks → Zustand store (`useStore`) → `MapView` renders SVG with d3 projections. Panels (`LeftPanel`, `DetailPanel`) read from same store for search/filter/selection.

### Key Files
- `src/types/index.ts` — All type definitions (`TrackedEntity`, `EntityType`, `MapMode`, etc.)
- `src/stores/useStore.ts` — Global state: entities, selection, map state, filters, theme
- `src/components/map/MapView.tsx` — Core visualization: projections, entity markers, trails, tooltips, viewport culling
- `src/lib/geo.ts` — `gcInterp()` for great-circle interpolation, `isGlobeVisible()` for 3D culling
- `src/lib/opensky.ts` — API client, state vector parser (raw arrays with hardcoded field indices)
- `src/lib/generate.ts` — Simulated routes, ports, stations, entity generation
- `src/lib/theme.ts` — Dark/light design tokens (colors, glows, entity-type colors)
- `src/lib/format.ts` — Speed/altitude/time formatters, type-to-color/icon helpers

### Store Pattern
Zustand store uses direct `set()` mutations with `setX` setter functions — no reducers or action types. State includes: entity arrays, `selectedId`/`hoveredId`, `mapMode` ("2d"/"3d"), `rotation`, `zoom`, `theme` ("dark"/"light"), filters, `isLive` toggle.

## Code Conventions

- Path alias: `@/*` maps to `./src/*`
- Entity markers in MapView use SVG paths/shapes per type (plane, ship, rectangle)
- Viewport culling filters entities outside visible area + behind globe in 3D mode
- Trail arrays are capped (15 points for vessels, 8 for trains)
- OpenSky status logic: on_ground → "On Ground", vertical_rate > 2 → "Climbing", < -2 → "Descending", altitude > 9000ft → "Cruising", else "En Route"
- Fonts: DM Sans (UI), JetBrains Mono (data labels)
